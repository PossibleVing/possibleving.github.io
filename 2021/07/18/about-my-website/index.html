你好鸭，我是 PossibleVing ~


我，一个啥也不懂的小白，终于花了两天时间，搭建了我的个人博客，由 Hexo 强力驱动，使用 Ayer 作为主题。

下面，我想记录一下我的折腾历程~

起因

其实我早就有建站的想法了，但是我清楚地认识到一个事实，那就是我的技术不达标。
举个例子：我们班上有个技术大佬（我这次构建博客也找他交流过技术），在听说我不会 HTML 之后，是这样说的：

好家伙，合着我应该是还没出生？！

出于对我技术能力的自知之明，因此建站这事儿我都准备放在大学再研究了。

后来有天，杨总（因为上文所述的这位大佬姓杨，我们都叫他“杨总”）有了一个 Hexo 博客：https://testgifts.tech/
很快啊，我啪的一下就点进去了[doge]，看上去很漂亮~

截图上是他已经切换到 Typecho 的网页。

所以呢，我萌生了做个博客的想法。但也只是“想法”而已。至少在我的固有思维中，建一个个人博客，是一件很麻烦的事情。
不过这两天我放假了。尽管高二升高三的暑假很短，但我想趁这段时间试试。一试才知道，搭建个 Hexo 博客并不难。

经过

一开始，我是知道可以在 GitHub Pages 上做静态网页的，只是不清楚怎么弄。所以，我参考了一个“简单”的替换网页教程：不需懂代码，10分钟在Github搭建免费且已成型的个人博客网站。
但后来发现有亿点麻烦：要替换的东西太多了。而且呢，做出来的东西也不是我想要的风格。于是我打算从头搞一个。

以下是错误示范！

我也不知道为什么（救命，我现在也很迷惑……），一开始就在 GitHub 上找了一个主题 Material，这是一个 Material Design 风格的 Hexo 主题。

还不错对吧？
问题就出在我直接把这个主题 Fork 到我的 Pages 仓库下，然后就准备打开网页……
……
笑死，根本没有网页。。。404，懂？

和杨总的仓库一对比，给我的直观感受是：
“哦，好像是没有 index.html 呢！”

对于当时我这个门外汉来说，Hexo 的主题配置文件夹和 /Public 目录是真的没啥区别。。。

然后，你们应该以为我是去百度解决方案了吧？
然鹅并没有。相反的是我思路清奇，找到原作者的 demo 网页，右键-查看页面源代码；再在 GitHub Pages 仓库根目录下新建一个 index.html。两边一复制，一粘贴：哈，我真是个小天才！
结果是这样的：

这波啊，这波我看啊，是意料之中。问了杨总，他说：
“我能说，你这个，就html文件生效了吗？”
呃，好像……是的。

到最后，我也没折腾出来。这时候我都不好意思问杨总了，只能再问问我能不能把他的博客再 Folk 一份。
杨总终于忍不住了：

对啊，我咋没想到呢？！
一搜，果不其然，网上一大堆教程

以下是正确示范~

之前我在折腾一些奇怪的东西（比如 Linux）就会问杨总一些问题，有很多次他都说让我去百度。我一开始以为他在敷衍我（逃~），后来才知道有的东西一搜就有。
所以这个故事告诉我们，有些看似不好解决的技术难题，其实网上都有解决方法，一定要善用搜索。我曾混迹于深度科技论坛，见过太多人把一些早就有解决方法的问题反复发帖求助，引发其他用户不满。
除了搜索，读技术文档也是需要的。上网求助并不意味着我们就要做个“伸手党”，有些操作背后的原理也应该吃透。
技术这东西，学到了，才是自己的。

emmm说了些废话。眼下，我已经配置好了 git，node.js 和 Hexo 环境。hexo init，hexo clean，hexo g，hexo s，hexo d 等一顿操作猛如虎~ 结果生成 Public 文件夹时，我的那个 Material 主题报错，是 materialcdn 出了问题。

一顿操作猛如虎，定睛一看，原地杵！——电影《飞驰人生》

Material 的官方文档比较笼统， Troubleshooting 比较少。一看 Issues 提这个问题的也挺多，但解决方法我试了，在我这边都没用。

咋办呢？我换了个主题——Ayer。
对就是大家现在看到的这个。

在这个主题下，配置稳如老狗[doge]。经过一系列个性化设置后，我的个人博客就诞生了！~

尾声

至此，我的博客就建好了。对于网站主体，我一行代码都没敲~

很快我就要上高三，这一年博客应该不会有太多新内容。
我本人对 UI 设计比较感兴趣，在深度科技论坛上我曾发过一些 PPT SHOW（就是拿 PowerPoint 做 UI）；对于 Linux，我也有一些了解和浅见。上大学以后，我会把我的一些想法分享在博客上，敬请期待~
